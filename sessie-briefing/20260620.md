### 2. API integraties voorbereiden (T0-008)

**Open-Meteo** (historische weerdata, gratis, geen key):
```javascript
// Voorbeeld aanroep
fetch(`https://archive-api.open-meteo.com/v1/archive
  ?latitude=50.47&longitude=5.67
  &start_date=2026-06-15&end_date=2026-06-15
  &daily=temperature_2m_max,temperature_2m_min,precipitation_sum,wind_speed_10m_max
  &timezone=Europe/Brussels`)
```

**OpenStreetMap Nominatim** (locatienaam uit coördinaten, gratis):
```javascript
// Voorbeeld aanroep
fetch(`https://nominatim.openstreetmap.org/reverse
  ?lat=50.47&lon=5.67&format=json`)
```

**Anthropic API** (AI-tekst generatie, betaald per gebruik):
```javascript
// Gebruiker voert API-key in bij openen creator
// Key wordt NIET opgeslagen — alleen in geheugen tijdens sessie
fetch("https://api.anthropic.com/v1/messages", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    model: "claude-sonnet-4-6",
    max_tokens: 1000,
    messages: [{ role: "user", content: prompt }]
  })
})
```

### 3. Route creator (T1-006)

**Workflow:**
1. GPX uploaden → automatisch berekend: afstand, duur, hoogtemeters op/neer, gemiddelde/max snelheid, hoogste/laagste punt, steilste stukken, rustpunten
2. Datum + locatie (auto via Nominatim uit GPX) → weerdata opgehaald via Open-Meteo
3. Foto-URLs invoeren (Cloudinary links)
4. Steekwoorden/ervaringen invoeren
5. AI genereert (optioneel): verhaal, tips, grid-samenvatting, foto-captions, vertaling
6. Live preview naast formulier
7. Export → downloadt `[id].json`

**Twee modi:**
- Handmatig — geen AI
- AI-assisted — AI stelt voor, gebruiker past aan

**Uitgebreid JSON schema (toevoeging op bestaand schema):**
```json
"status": "draft | published",
"planned_date": "2026-07-10",
"published_date": "2026-07-12",
"source_reference": "https://www.alltrails.com/trail/...",
"gpx_stats": {
  "distance_km": 12.3,
  "duration_hours": 3.5,
  "elevation_up_m": 340,
  "elevation_down_m": 310,
  "avg_speed_kmh": 3.8,
  "max_speed_kmh": 6.2,
  "highest_point_m": 520,
  "lowest_point_m": 180
},
"weather": {
  "date": "2026-06-15",
  "temperature_min": 12,
  "temperature_max": 22,
  "precipitation_mm": 0,
  "wind_kmh": 8,
  "condition": "zonnig",
  "source": "Open-Meteo"
}
```

### 4. Route detail pagina (T1-002)

**Structuur:**
topbar → hero foto → stats blok (GPX stats) → weer-blok → verhaal-blokken → tips → "Bekijk kaart" knop → "Print / Bewaar als PDF" knop → footer

**Print:**
- `@media print`: A4-formaat (794px), kaart → statische afbeelding, alle links actief
- WYSIWYG: pagina ziet er op scherm hetzelfde uit als bij afdrukken
- Print-knop is publiek toegankelijk

## Architectuurafspraken

| Onderwerp | Afspraak |
|-----------|---------|
| **Foto-opslag** | Cloudinary — directe URL in JSON, geen lokale kopie |
| **Kaart** | Apart tabblad (`[id]-map.html`) |
| **Print** | Browser-native, geen externe library |
| **Login creator** | Geen in MVP — uitgesteld naar T6-001 |
| **Navbar** | Definitief verwijderd |
| **AI API-key** | Ingevoerd door gebruiker, niet opgeslagen |
| **Weerdata** | Open-Meteo, gratis, geen key |
| **Locatienaam** | Nominatim, gratis, geen key |
| **i18n** | Elke nieuwe pagina krijgt eigen namespace in `data/i18n/nl/` en `data/i18n/en/` |
| **Scripts volgorde** | i18next CDN → `i18n.js` → `app.js` → pagina-JS |
