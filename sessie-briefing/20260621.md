### Nog aan te maken vóór sessie 04

**1. profiles tabel** — via Supabase dashboard → SQL Editor:
```sql
create table profiles (
  id uuid references auth.users primary key,
  username text,
  is_admin boolean default false,
  created_at timestamptz default now()
);
```

**2. Trigger voor automatisch profiel aanmaken bij registratie:**
```sql
create or replace function public.handle_new_user()
returns trigger as $$
begin
  insert into public.profiles (id, username)
  values (
    new.id,
    coalesce(new.raw_user_meta_data->>'username', split_part(new.email, '@', 1))
  );
  return new;
end;
$$ language plpgsql security definer;

create trigger on_auth_user_created
  after insert on auth.users
  for each row execute procedure public.handle_new_user();
```

**3. Eigen account admin maken** — na registratie via de site:
```sql
update profiles set is_admin = true where username = 'jouw_gebruikersnaam';
```

## Architectuurafspraken (bevestigd sessie 03)

| Onderwerp | Afspraak |
|-----------|---------|
| **Script volgorde** | Altijd onderaan `<body>`: Eruda → Supabase SDK → i18next → i18n.js → auth.js → topbar-auth.js → app.js → [pagina].js |
| **Eruda** | Niet verwijderen als aanwezig in HTML pagina |
| **CSS** | Altijd in `<head>` |
| **Favicon** | Elke nieuwe pagina krijgt favicon tags na `<title>`, vóór CSS |
| **Foto-opslag** | Cloudinary — directe URL in JSON |
| **AI API-key** | Ingevoerd door gebruiker, niet opgeslagen |
| **Weerdata** | Open-Meteo, gratis, geen key |
| **Locatienaam** | Nominatim, gratis, geen key |
| **Admin toegang** | Via `is_admin = true` in profiles tabel |
